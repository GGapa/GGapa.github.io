为了强化笔者对LCA的理解，故作此文。\n

1 DFS序求LCA\n

1.1 算法介绍\n

考虑树上的两个节点 $u$, $v$ 和其祖先 $d$，我们之所以使用欧拉序求解 LCA 是因为在欧拉序中 $d$ 一定在 $u,v$ 之间出现。但对于 DFS 序来说，$d$ 一定在 $u,v$ 之前出现。\n

令 $u$ 的 DFN 小于 $v$，且 $u\\ne v$ 。\n

当 $u，v$ 没有祖先关系时。在 DFN 中最先出现的是 $d$ ，其次是 $u$，最后是 $v$ 。设 $v'$ 是 $d$ 的儿子。根据 DFS 的特性可知，$v'$ 在 $u\\sim v$ 的 DFN 序之间。\n

这意味着我们只需要求出 $u \\sim v$ 的 DFS 序之间深度最小的一个节点，那么他的父亲便是我们苦苦寻找的 LCA。\n

如何保证这个方法的正确性？在 $u\\sim v$ 的 DFS 序之间，$d$ 一定不会存在，而 $d$ 的儿子一定存在。\n

当 $u, v$ 存在祖先关系，这更容易判断了，根据定义， $u$ 一定是 $v$ 的祖先。查询的序列由 $[dfn_u,dfn_v]$ 变成 $[dfn_u+1,dfn_v]$ 。为什么要 "$+1$"？根据上一种情况可知，此时的祖先是 $u$ ，但此时深度最小的节点一定是 $u$，如果在这时取它的父亲，很明显不是我们要找的正确答案。\n

很显然，当$u，v$ 没有祖先关系时， $u$ 一定不等于 $d'$ ，所以刚刚提出的结论适用 $u$ 的 DFN 小于 $v$，且 $u\\ne v$ 所有的情况。\n

若 $u=v$ 时，它们的 LCA 一定是 $u$ ，这是唯一一种需要特判的情况。\n

综上所述，若 $u\\ne v$ ，$u,v$ 的 LCA 是 $[dfn_u,dfn_v]$ 中深度最小的点的父亲，若 $u=v$ 时，$u,v$ 的 LCA 是 $u$。\n

1.2算法实现\n

可以通过 ST 表来查询 $[dfn_u,dfn_v]$ 中深度最小的点。时间复杂度 $O(nlogn)$ 。\n

以下是模板题目 P3379 的代码：\n

\/\/ P3379 【模板】最近公共祖先（LCA）\n\/\/ https:\/\/www.luogu.com.cn\/problem\/P3379\n\/\/ 2023-11-09 21:40\n#include <iostream>\n#include <vector>\n#include <cmath>\nusing namespace std;\nconst int maxn = 500000 + 5;\n\nint n, m, s;\nvector<int> G[maxn];\nint dfn[maxn], mi[20][maxn];\n\nint get(int x, int y) {\n    return dfn[x] < dfn[y] ? x : y;\n}\n\nvoid dfs(int x, int f) {\n    dfn[x] = ++dfn[0]; mi[0][dfn[x]] = f;\n    for(auto to : G[x]) if(to != f) dfs(to, x);\n}\n\nint lca(int x, int y) {\n    if(x == y) return x;\n    if((x = dfn[x]) > (y = dfn[y])) swap(x, y);\n    int d = log2(y - x++);\n    return get(mi[d][x], mi[d][y - (1 << d) + 1]);\n}\n\nint main() {\n    ios::sync_with_stdio(0);\n    cin.tie(0);\n    cout.tie(0);\n    \/\/ freopen("AKIOI.in", "r", stdin);\n    \/\/ freopen("AKIOI.out", "w", stdout);\n    cin >> n >> m >> s;\n    for(int i = 1, u, v; i < n; i++)  {\n        cin >> u >> v;\n        G[u].push_back(v); G[v].push_back(u);\n    }\n    dfs(s, 0);\n    for(int i = 1; i <= log2(n); i++) for(int j = 1; j + (1 << i) - 1 <= n; j++) \n        mi[i][j] = get(mi[i - 1][j], mi[i - 1][j + (1 << i - 1)]);\n    while(m--) {\n        int x, y;\n        cin >> x >> y;\n        cout << lca(x, y) << "\\n";\n    }\n    return 0;\n}\n\n\/*\n编程语言\nC++14 (GCC 9)\n代码长度\n1.25KB\n用时\n1.64s\n内存\n77.61MB\nGGapa\n\n所属题目\nP3379 【模板】最近公共祖先（LCA）\n评测状态\nAccepted\n评测分数\n100\n提交时间\n2023-11-09 22:02:41\n*\/\n
2 树链剖分 LCA\n
2.1 算法介绍\n



对于一个父节点的所有儿子来说，在它们之间子树大小最大的便是重儿子($hson$)，其余的儿子是轻儿子。我们可以对一棵树上所有的非叶子节点进行操作，求出他们的重儿子。从一个节点 $u$ 出发，每次递归到下一个重儿子直到叶子节点未知，可得一个递归函数 $f(x)=f(hson_x)$ 。在递归的过程中访问到的节点可以连成一条链，这条链便是重链。每条重链的出发点被称为 $top$ 。\n
考虑树上的两个节点 $u$, $v$ 和其祖先 $d$，\n

    \n
  1. 若 $top_u = top_v$ 这说明 $u,v$ 在一条重链上，$u,v$ 一定存在祖先关系，此时 $u,v$ 中深度较小的节点便是他们的 LCA。\n
    2. 

  2. 否则，需要把深度较大的节点跳到这条重链的顶端。并重复执行操作1。\n\n
    为什么不能同时将 $u, v$  上跳？令 $u$ 的深度大于 $v$ 因为有时深度较大的节点的 $top$ 便是 $v$ 所在的重链，每跳一次都会使它离开当前的重链。这时 $v$ 往上跳就不能达成条件，效率低下。\n

    时间复杂度 $O(n\\log n)$\n

    2.2 算法实现\n
    

    需要进行两次递归，第一次递归求出每个非叶子节点的重儿子；第二次循环求出 $top$，也就是重链。\n

    以下是模板题目 P3379  的代码：\n
    \/\/ P3379 【模板】最近公共祖先（LCA）\n\/\/ https:\/\/www.luogu.com.cn\/problem\/P3379\n\/\/ 2023-11-09 20:41\n#include <iostream>\n#include <vector>\nusing namespace std;\nconst int maxn = 500000 + 5;\n\nint n, m, s; \nvector<int> G[maxn];\nint si[maxn], top[maxn], hson[maxn], fa[maxn], dep[maxn];\n\nvoid dfs1(int x, int f) {\n    si[x] = 1; dep[x] = dep[f] + 1; fa[x] = f;\n    for(auto to : G[x]) {\n        if(to == f) continue;\n        dfs1(to, x);\n        si[x] += si[to];\n        if(si[to] > si[hson[x]]) hson[x] = to;\n    }\n}\n\nvoid dfs2(int x, int tp) {\n    top[x] = tp;\n    if(!hson[x]) return ;\n    dfs2(hson[x], tp);\n    for(auto to : G[x]) {\n        if(to == fa[x] || to == hson[x]) continue;\n         dfs2(to, to);       \n    }\n}\n\nint lca(int x, int y) {\n    while(top[x] != top[y]) {\n        if(dep[top[x]] < dep[top[y]]) swap(x, y);   \n        x = fa[top[x]];\n    }\n    return (dep[x] < dep[y] ? x : y);\n}\n\nint main() {\n    ios::sync_with_stdio(0);\n    cin.tie(0);\n    cout.tie(0);\n    \/\/ freopen("AKIOI.in", "r", stdin);\n    \/\/ freopen("AKIOI.out", "w", stdout);\n    cin >> n >> m >> s;\n    for(int i = 1, u, v; i < n; i++) {\n        cin >> u >> v;\n        G[u].push_back(v);\n        G[v].push_back(u);\n    }\n    dfs1(s, 0);\n    dfs2(s, s);\n    while(m--) {\n        int x, y;\n        cin >> x >> y;\n        cout << lca(x, y) << "\\n";\n    }\n    return 0;\n}\n\n\/*\n编程语言\nC++14 (GCC 9)\n代码长度\n1.37KB\n用时\n1.86s\n内存\n55.85MB\nGGapa\n\n所属题目\nP3379 【模板】最近公共祖先（LCA）\n评测状态\nAccepted\n评测分数\n100\n提交时间\n2023-11-09 21:04:30\n*\/\n
    3 倍增 LCA\n
    3.1 算法介绍\n
    


    相对于暴力算法，倍增 LCA 减少了向上跳的次数，如何减少？每次从大到小尝试跳 $2^i$ 个节点，直到到达目的地。\n

    为了方便跳，需要记录每一个节点 $2^i$ 级的祖先。\n

    考虑树上的两个节点 $u$, $v$，需要先使 $dep_u=dep_v$\n

    接着，两个节点同时往上跳，直到$fa{vi}=fa{ui}$为止，此时 $fau$ 便是 $u,v$ 的 LCA。有一点需要注意，要跳到 $fa{ui}=fa_{vi}$ 而不是 $u=v$ 是为了防止跳过了，找到的是他们的公共祖先，而不是 LCA。\n
    时间复杂度$O(n\\log n)$\n

    3.2 算法实现\n
    

    以下是模板题目 P3379  的代码：\n
